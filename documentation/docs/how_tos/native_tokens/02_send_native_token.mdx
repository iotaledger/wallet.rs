After you have [minted](01_mint_native_token.mdx) a
[native tokens](https://wiki.iota.org/introduction/develop/explanations/what_is_stardust/tokenization#native-tokens),
you can easily send it by calling the `Account.send_native_tokens(addresses_native_tokens, options)` function.

As with any output, you can set a
[storage deposit](https://wiki.iota.org/introduction/develop/explanations/what_is_stardust/storage_deposit) and
[output unlock conditions](https://wiki.iota.org/introduction/develop/explanations/what_is_stardust/unlock_conditions).
Keep in mind that if you set unlock conditions, whoever you send the native tokens to may need to
[claim them](../outputs_and_transactions/04_claim_outputs.mdx).
